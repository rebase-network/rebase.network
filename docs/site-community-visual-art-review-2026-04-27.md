# Rebase.network 整站视觉效果与艺术风格复审

日期：2026-04-27

审阅对象：当前本地真实页面 `http://127.0.0.1:4321`

参考：

- `apps/web/design_principles.md`
- `apps/web/DESIGN.md`
- `docs/site-redesign-followup-implementation-results-2026-04-27.md`

目标：判断当前站点是否足够贴近“社区媒体、温暖、活跃、有人味”，并找出仍显严肃、产品化或视觉过重的问题。

## 总体结论

第一轮与二次整改后，站点已经明显从“严肃产品官网”转向了“温暖的社区媒体站点”。页面背景、卡片、浅色侧栏、RSS 入口和贡献者模块都比早期更轻。

但当前仍有几类问题会削弱社区感：

1. 首页仍有部分 dashboard 和大型 landing page 视觉痕迹。
2. 贡献者测试数据直接显示 `test ...`，会破坏真实社区氛围。
3. 招聘卡片仍偏表格化，薪资、模式、地点像企业招聘面板。
4. GeekDaily 筛选区存在不必要的大块空白，像工具页而不是轻归档。
5. 详情页 hero 字号和行高仍偏正式，尤其长英文活动标题显得像会议官网。
6. 移动端导航可发现性不足，部分导航项在首屏被截断。
7. 事件卡片默认封面在移动端 full-page 截图中容易出现大块空白感，削弱活动氛围。

本轮不需要推倒重来，建议继续做“社区化微调”：减少仪表盘感、去除测试痕迹、降低视觉重量、提高移动端浏览舒适度。

## 页面问题清单

### 首页

问题：

- Hero 区仍有大型 `REBASE` 水印，视觉上更像品牌 landing page，而不是社区公告入口。
- 三个统计卡片 `12+ / 13+ / 7` 独立成大卡，接近 dashboard 指标，和“社区正在发生什么”的人味不够一致。
- 招聘卡片在首页仍以薪资/模式/地点三列表格呈现，偏正式、偏招聘平台。
- 移动端首页的活动卡片如果封面未即时加载，会出现大块浅色空白区域。

影响：

- 首页整体虽然温暖，但“品牌官网”和“社区媒体首页”的边界仍有些摇摆。
- 指标卡和招聘卡会让页面偏产品化。

优先级：P1

### GeekDaily

问题：

- 搜索卡片与筛选卡片同一行等高，搜索输入下方出现大面积空白。
- Hero 上仍保留 `GEEKDAILY` 大水印，虽然很淡，但和社区归档页的轻量目标有轻微冲突。
- 列表卡片信息清晰，但整体仍偏规整，缺少“日报编辑台”的轻微手作感。

影响：

- 搜索筛选区像后台工具面板，不像轻归档。
- 首屏信息密度被空白稀释。

优先级：P1

### 文章与活动详情页

问题：

- 文章详情 hero 标题仍非常大，长中文标题占据过多注意力。
- 活动详情长英文标题换行很硬，视觉上偏会议官网/产品发布页。
- 详情正文卡片在内容较短时会因为 grid 拉伸出现大量空白。

影响：

- 详情页阅读感已经改善，但仍偏正式。
- 空白过多会让页面显得像未填完整的模板。

优先级：P1

### 招聘

问题：

- 招聘列表卡片仍是明显三列表格结构。
- 薪资信息视觉权重偏高，容易让页面变成招聘平台，而不是社区发布板。
- `Who-Is-Hiring` 英文栏目名在导航和标题中仍保留，和中文社区语境略有距离。

影响：

- 招聘页比其他栏目更严肃，更工具化。
- 读者更容易把它理解为招聘产品，而不是社区成员提交的信息板。

优先级：P1

### 贡献者

问题：

- 新增测试智囊团成员直接显示 `test geekdaily advisor 01` 等名称。
- 志愿者分组仍显示 `test volunteer 01` 等名称。
- 贡献者卡片头像 fallback 的阴影和卡片 hover 仍略重。
- 大量测试成员如果使用同样 `test` 文案，会显得页面是假数据，而不是真实社区。

影响：

- 这是当前最明显破坏“社区感”的问题。
- 人物页最需要真实感，不能让用户第一眼看到测试痕迹。

优先级：P0

### 移动端

问题：

- Header 导航在 390px 宽度下只显示前几项，后续入口可横向滚动但不够明显。
- Hero 标题仍偏大，移动端首屏信息量有限。
- 长页面卡片之间已经可读，但局部模块仍像一组白色表单卡。

影响：

- 移动端用户可能不知道还有“活动、贡献者、关于”等导航入口。
- 页面温暖感有了，但轻松浏览感还可以继续增强。

优先级：P1

## 本轮建议方向

1. 去除测试痕迹，让贡献者测试数据也像真实社区成员。
2. 降低首页 dashboard 感，把统计卡片变成轻量社区小记。
3. 修 GeekDaily 搜索筛选区域空白。
4. 降低招聘卡片的表格感，把岗位事实改成更轻的横向信息片。
5. 调整详情页 hero 字号、行高和 grid 对齐，减少模板感。
6. 改善移动端导航横向滚动的可发现性。
7. 让活动卡片默认封面更稳定地呈现，减少移动端空白感。

## 不建议本轮做的事

- 不重新设计 logo 或品牌主色。
- 不引入插画系统或复杂贴纸风视觉。
- 不改后台数据模型。
- 不做大规模页面重排。
- 不新增截图视觉回归系统，只补充 Playwright smoke 和人工截图检查。
