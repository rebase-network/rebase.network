# Rebase.network 视觉风格整改方案

日期：2026-04-27

依据：`docs/site-visual-style-review-2026-04-27.md`

目标：让整站视觉更贴近社区，更松弛、更有人味，同时保持 Rebase 作为开发者社区媒体的可信和可读。

## 设计方向

本轮视觉整改不做全站重设计，而是做“社区化收敛”：

- 从精致产品站，转向社区公告板和内容归档。
- 从强卡片和大阴影，转向轻边框、浅底色、自然分隔。
- 从宣言式 hero，转向短栏目名和真实内容入口。
- 从 dashboard 式侧栏，转向阅读辅助信息。
- 从抽象品牌表达，转向真实的人、内容、活动和更新。

## 视觉原则

### 1. 默认轻，重点才重

普通列表卡片、工具栏、信息块默认轻量。只有首页主介绍、当前活动、最新文章等真正需要聚焦的区域保留较高视觉权重。

### 2. 中文信息自然读

中文站点不应大量使用 uppercase 和过大的 letter spacing。日期、作者、状态、栏目标签可以保持小字号，但语气应更自然。

### 3. 卡片像社区条目，不像产品卡片

卡片 hover 不应普遍大幅上浮。普通内容卡片只需要轻微边框变化和浅背景变化。

### 4. 人比装饰重要

贡献者头像、编辑、推荐人、活动组织者、投稿入口等信息应比英文水印和装饰色块更重要。

### 5. 列表页先给内容

文章、活动、招聘、GeekDaily 等列表页应该快速进入真实内容，hero 不再承担大段品牌表达。

## 整改项

### VS-01 全局视觉 token 收敛

影响文件：`apps/web/src/styles/global.css`

方案：

- 降低 `--shadow` 和 `--shadow-strong` 的强度。
- 为普通卡片增加更轻的 `--shadow-soft`。
- 降低 `.card` 默认 `backdrop-filter`，或只保留轻 blur。
- 将默认 hover 位移从 `-4px` 收敛到 `-1px` 或 `-2px`。
- 保留暖纸质背景，但降低网格和径向色洗的存在感。

验收：

- 页面不再大量出现漂浮玻璃卡片感。
- 内容块仍然有边界和层级。
- 首页、列表页、详情页视觉更轻。

### VS-02 Header 与 SectionHeader 轻量化

影响文件：

- `SiteHeader.astro`
- `SectionHeader.astro`

方案：

- Header 阴影更轻，背景更接近半透明纸面。
- 导航 active 状态保留，但 hover 不明显上浮。
- SectionHeader 右侧链接改为轻量链接或弱按钮，去掉重阴影。

验收：

- 导航仍清晰，但不会像 SaaS 顶栏。
- 每个 section 的“查看更多”不再像强 CTA。

### VS-03 Hero 系统社区化

影响文件：

- `ListPageHero.astro`
- `articles/index.astro`
- `events/index.astro`
- `contributors.astro`
- `who-is-hiring/index.astro`
- `geekdaily/index.astro`

方案：

- 栏目页 H1 使用短栏目名。
- 摘要保留，但控制在 1-2 个自然段。
- 默认移除英文水印，或极度弱化。
- Hero 卡片阴影减少，内部留白收敛。
- 未上线入口不以按钮形态出现。

验收：

- 移动端 H1 不超过 2 行。
- 桌面首屏能更快看到内容列表。
- 页面不像营销落地页。

### VS-04 卡片系统社区条目化

影响文件：

- `ArticleCard.astro`
- `EventCard.astro`
- `GeekDailyCard.astro`
- `JobCard.astro`
- `ContributorCard.astro`

方案：

- 普通卡片 hover 降低位移和阴影。
- 元信息不强制 uppercase。
- tag、chip、状态标签改成浅色小标识。
- EventCard 修正历史活动 CTA 文案。
- JobCard 修正公司占位值显示。

验收：

- 列表更像内容归档和社区帖子。
- 卡片仍可点击，但不强商业化。
- 长标题、缺图、缺公司名仍稳定展示。

### VS-05 首页动态区改为社区公告板

影响文件：`index.astro`

方案：

- 将“社区正在发生什么”从深色或重面板改为浅色公告板。
- Live wire item 使用轻分隔、栏目标签和日期。
- Stats strip 弱化为社区信号，减少背书感。
- 贡献者预览保留照片墙感，但降低装饰文案。

验收：

- 首页更像正在更新的社区，而不是品牌展示页。
- 读者能快速看到最近内容和社区动态。

### VS-06 详情页浅侧栏与阅读优先

影响文件：

- `global.css`
- `articles/[slug].astro`
- `events/[slug].astro`
- `who-is-hiring/[slug].astro`
- `geekdaily/[slug].astro`

方案：

- 详情侧栏改为浅底辅助区。
- 减少深色背景、强阴影和 dashboard 感。
- 去掉首屏附近重复元信息。
- 保留一个主操作，其余使用轻链接。

验收：

- 正文更早出现。
- 侧栏信息可读但不抢正文。
- 活动报名、文章 RSS、招聘投递等操作清晰但不过重。

### VS-07 贡献者页真实成员感

影响文件：

- `contributors.astro`
- `ContributorCard.astro`
- `lib/content.ts` 如需前端去重

方案：

- 贡献者页首屏拆成短模块和加入入口。
- 同一成员尽量合并展示，或减少重复造成的视觉疲劳。
- 缺头像时使用统一默认样式，而不是强烈的单字母色块。
- 历史贡献者更紧凑。

验收：

- 页面更像真实社区成员墙。
- “如何加入”更清楚。
- 占位头像不成为主要视觉噪声。

## 不做事项

- 不引入新的 UI 框架。
- 不重做 logo 和品牌资产。
- 不建设复杂动画系统。
- 不把站点改成游戏化或社交网络风格。
- 不为每个页面单独设计完全不同的视觉语言。

## 推荐实施顺序

1. 全局 token 与卡片 hover 收敛。
2. Header、SectionHeader、普通卡片轻量化。
3. 栏目页 hero 短标题和水印收敛。
4. 首页动态区和 stats 弱化。
5. 详情页浅侧栏。
6. 贡献者页和历史归档体验优化。

## 成功标准

- 站点仍然有 Rebase 的暖色和编辑感。
- 视觉不再像严肃产品官网。
- 列表页更紧凑，详情页更可读。
- 首页更像社区正在持续发生事情。
- 贡献者和内容来源更有存在感。
