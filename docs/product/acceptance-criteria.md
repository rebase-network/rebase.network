# 当前验收基线

这个文件定义当前仓库中“什么算通过验收”。

它只保留仍然适用于今天开发和发布的验收要求，不重复历史范围说明，也不替代执行层面的 QA 手册。

执行方式见 `docs/operations/quality-assurance.md`。

## 核心验收门槛

一项改动在合并或发布前，至少要满足它所影响范围内的以下门槛：

### 1. 数据与契约正确

- 改动后的数据结构仍与 `docs/product/content-model.md` 一致
- 状态、字段、路由和发布约束没有被悄悄破坏
- 公共网站只读取应公开的已发布内容

### 2. 公共界面可用

如果改动影响公共页面：

- 受影响路由能返回正确内容
- 关键链接、分页、搜索、RSS 或详情跳转行为正常
- 空状态、缺失媒体和长内容不会破坏页面
- 桌面端和移动端都完成了实际浏览器检查

### 3. 管理工作流可完成

如果改动影响管理工作台或 API 写路径：

- 对应列表页、编辑页或设置页可正常进入
- 校验提示明确、可执行
- 保存、发布、归档或预览链路在受影响模块中可完成
- 权限相关改动能正确影响导航和操作可见性

### 4. 运维与发布可落地

如果改动影响运行时、部署或生产配置：

- 健康检查、就绪检查和关键路由仍然成立
- 对应的部署或 dry-run 命令仍然匹配文档
- 相关事实文档已经同步更新

## 受影响界面的最低验收要求

### 公共内容页面

适用于首页、About、招聘、GeekDaily、文章、活动、贡献者和页脚：

- 页面展示正确内容
- 路由与跳转符合当前 URL 规则
- 元数据、列表顺序和空状态合理
- 如存在外部动作链接，其行为清晰且可用

### RSS 与发现类输出

适用于 RSS、`robots.txt`、`sitemap.xml`：

- 输出格式有效
- 链接目标符合当前公共路由约定
- 内容来源仍然只来自应公开的已发布内容

### GeekDaily 搜索

如果改动影响 GeekDaily 搜索：

- 搜索入口存在
- 结果能链接到正确期目
- 空结果、加载中和空闲状态可区分

### 管理工作流

适用于内容、媒体、站点设置、工作人员与权限：

- 列表可加载
- 编辑器可保存
- 需要校验的字段会被明确阻止或提示
- 如果涉及发布内容，对应公共页面能看到结果

## 发布门槛

一批改动可以进入合并或发布，前提是：

- 受影响范围已经按 `docs/operations/quality-assurance.md` 完成验证
- 没有未记录的已知回归
- 所有文档更新都反映当前事实，而不是历史计划
