# 2026-06 仓库审阅记录

这份文档记录 2026-06-09 对 `rebase.network` 仓库进行的一次文档、代码和依赖审阅。

它只保留当前仍然值得跟踪的发现与修复状态，不替代长期事实文档。

## 本次发现

### 1. 文档与实现不一致

- `docs/product/content-model.md` 曾记录 `/who-is-hiring` 支持 `q`、`location` 和 `mode` 查询参数。
- 当前实现只支持招聘列表与详情，不支持这些筛选查询。
- 这类差异会误导产品评审、测试范围和后续实现判断。

### 2. 媒体“私有可见性”不是真正的访问控制

- 媒体上传当前写入公共 R2 路径，并生成可直接访问的公开 URL。
- 先前的 `visibility=private` 更像数据库元数据，而不是实际的访问限制。
- 在没有私有桶、签名 URL 或受控下载代理之前，不应继续把它作为可用能力暴露给运营端。

### 3. 上传链路缺少服务端硬限制

- 上传入口此前没有明确的文件大小上限。
- 服务端此前会在缺少足够前置限制时读取整份上传文件，再做后续处理。
- 上传能力此前缺少严格的 MIME 白名单与额外的内容级校验。

### 4. 依赖存在安全与维护性欠账

- `apps/api` 和 `apps/web` 的若干生产依赖落后于当前已修复版本。
- `pnpm audit --prod` 在审阅时报告了 Hono、Drizzle ORM、Astro、Vite、Better Auth 等相关公告。
- 仓库里还存在未被源码使用的运行时依赖，应优先清理而不是继续保留。

## 当前修复结果

- 已修正文档事实来源，移除了与当前实现不符的 `/who-is-hiring` 查询参数说明。
- 已停止在新写入路径中暴露“私有媒体”能力；当前上传与手动写入只接受公开媒体。
- 已为上传路径加入服务端 body limit、单文件大小上限和 MIME 白名单，并增加内容签名校验。
- 已清理未使用的运行时依赖，并升级高优先级生产依赖。
- 修复后重新执行 `pnpm audit --prod --json`，当前结果为 `0` 个生产依赖漏洞。

## 当前约束

- 媒体上传当前只支持公开媒体。
- 当前单文件上传上限为 `12 MiB`。
- 当前允许上传的 MIME 类型为：
  - `image/jpeg`
  - `image/png`
  - `image/webp`
  - `image/gif`
  - `application/pdf`
  - `video/mp4`

## 后续事项

- 如需真正的私有媒体能力，应引入私有桶、签名 URL 或受控下载代理，而不是继续复用公开 URL 路径。
- 历史记录里若仍存在 `private` 值，应视为兼容性残留，而不是当前可依赖的产品能力。
