# 文档归档

这个目录用于存放仍有参考价值、但不再作为当前事实来源的历史文档。

这里的文档可以保留背景信息、阶段性规划或已经完成的版本边界，但它们不应被当作当前实现、当前流程或当前优先级的事实来源。

## 使用规则

- 需要了解今天系统如何工作时，优先查看 `docs/architecture/`、`docs/operations/` 与 `docs/product/` 下的活跃文档
- 只有在需要追溯历史决策、已完成阶段范围或旧计划时，才查看这里的内容
- 如果一份文档仍然持续约束今天的开发、评审或发布，它不应该留在归档区
