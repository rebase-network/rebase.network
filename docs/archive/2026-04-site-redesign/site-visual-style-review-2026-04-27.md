# Rebase.network 整站视觉效果与艺术风格审阅

日期：2026-04-27

目标：围绕“整体风格更贴近社区一点，不要太严肃”的方向，审阅 `rebase.network` 当前真实页面和 `apps/web` 视觉实现，识别会让站点显得过于正式、产品化或营销化的问题，并提出可执行的视觉调整建议。

## 总体判断

当前站点已经有温暖、编辑化、内容站的基础，整体不是冷硬企业站。但从视觉气质看，它更接近“精致媒体站 / 产品宣传站”，还不够像一个由志愿者长期维护、真实成员持续参与的开发者社区。

造成这种感受的主要因素：

- 全站大量使用玻璃卡片、强阴影和上浮 hover，让内容像商业产品卡片。
- Hero 标题偏大、偏宣言化，首屏更像品牌表达，而不是社区内容入口。
- 多处英文水印、uppercase 元信息、letter spacing 和深色面板增强了正式感。
- 详情页和侧栏的深色信息面板偏像 dashboard，社区阅读感不足。
- “人”的视觉线索不够靠前，贡献者、编辑、推荐人、组织者没有形成足够自然的社区存在感。

更适合 Rebase 的方向不是“更可爱”或“更娱乐化”，而是更松弛、更日常、更像社区公告板和内容归档：保留温暖纸感和编辑气质，降低高冷精致感，让真实内容和真实的人站到前面。

## 页面级问题

### 首页

问题：

- Hero 卡片有较强阴影、水印和装饰层，视觉像品牌落地页。
- “社区正在发生什么”面板如果使用深色或强对比，会像控制台，不像社区动态。
- Stats strip 容易让首页变成“证明我们是谁”，而不是“社区最近在发生什么”。
- 各 section 的 CTA 都是较明显的 pill button，模块感和营销感偏重。

建议：

- 首页保留表现力，但降低 hero 阴影和水印权重。
- 将动态区做得更像 bulletin board，使用浅底、分隔线、轻标签。
- Stats 可以弱化为小型社区信号，不要占据强背书位置。
- Section 链接从强按钮降为轻量文本/浅按钮。

### 文章页

问题：

- H1 是长口号，页面更像宣传语入口，不像文章栏目。
- “推荐文章”是禁用按钮，带来未上线功能感。
- 最新文章 spotlight 信息量不足，视觉比普通卡片重，但内容判断信息不够。

建议：

- H1 改为短栏目名“社区文章”。
- 描述文案减少口号，直接说明可读内容与投稿入口。
- 未上线入口不要以按钮出现。
- 最新文章补摘要，或统一为一组文章列表。

### 活动页

问题：

- H1 长句偏品牌化，真实活动入口被推后。
- 历史活动卡片仍出现报名型操作，影响可信度。
- 历史活动一次性铺开，视觉密度和浏览成本都偏高。

建议：

- H1 改为“社区活动”。
- 活动状态和 CTA 先修正，再考虑年份筛选。
- 历史活动卡片压缩字段，降低视觉重量。

### GeekDaily 页

问题：

- 页头中“推荐内容 ↗”像可点击入口但不可用。
- 搜索与筛选区域卡片感较强，像工具台，和社区归档感还有距离。
- 分页为按钮增强实现，不利于基础可访问浏览。

建议：

- 删除或弱化未上线入口。
- 筛选区保留工具属性，但减少阴影和装饰。
- 分页改为真实链接，前端增强只作为补充。

### 招聘页

问题：

- 招聘频道本身需要可信，但视觉上与其他卡片一致，缺少“社区转发/请谨慎辨别”的语气差异。
- 长说明集中在 hero，首屏文字负担重。
- 职位卡片中公司、状态、远程、薪资等信息较硬，像招聘 SaaS。

建议：

- Hero 更直接：开放岗位、提交职位、订阅 RSS。
- 风险提示可以做成浅提示条，不塞进长段。
- 卡片更像社区转发帖：标题、公司、地点、更新时间、来源/投递方式清晰即可。

### 贡献者页

问题：

- 页头长文让“如何参与”不够醒目。
- 贡献者卡片较像资料卡，缺少轻松的照片墙/成员名片感。
- 大量占位头像和重复成员会削弱真实社区感。

建议：

- 首屏拆成“做什么 / 需要什么 / 如何加入”三块，并前置加入入口。
- 头像 fallback 统一克制，不让单字母成为主要视觉。
- 同一成员合并展示多角色，历史贡献者做紧凑归档。

### 详情页

问题：

- 文章、活动、招聘详情页侧栏视觉偏重，深色面板较严肃。
- Hero 和正文前的信息重复，让阅读像进入资料系统。
- 正文开始位置偏后。

建议：

- 详情页转向阅读页面：浅侧栏、轻分隔、少重复。
- 主 CTA 保留一个，其他入口做轻链接。
- 正文优先，元信息只保留决策所需内容。

## 组件级问题

### Card

问题：全局 `.card` 默认有玻璃模糊、动画、阴影和较明显 hover，导致普通内容都像精致产品卡片。

建议：降低默认卡片强度。普通卡片用浅底、细边框、小阴影或无阴影；重点内容再单独加突出样式。

### Hero

问题：多处 hero 使用大标题、英文水印、强阴影和装饰渐变，统一性强但社区感不够。

建议：列表页 hero 更紧凑，H1 直接使用栏目名；英文水印默认移除，必要时只在首页保留极弱品牌痕迹。

### Section Header

问题：右侧链接像 CTA button，所有模块都带“运营转化”感。

建议：改成轻量文本链接或弱按钮，减少阴影和上浮。

### Metadata

问题：uppercase、letter spacing、粗体 metadata 在中文内容中偏正式。

建议：中文页面元信息尽量用自然文本，不强制 uppercase；保留小字号和颜色层级即可。

### Header / Footer

问题：Header sticky card 与 pill nav 视觉较正式；Footer 当前结构可用，但仍是卡片式产品收尾。

建议：Header 保留清晰导航但减轻阴影；Footer 更像社区索引，社交入口和社区资源更自然。

## 艺术方向建议

目标关键词：

- 社区公告板
- 真实成员
- 内容归档
- 暖纸感
- 轻量编辑感
- 松弛但可信

保留：

- 暖色纸质背景。
- 柔和边框和浅色 surface。
- Serif 标题作为少量编辑化强调。
- 内容卡片网格和清晰导航。

收敛：

- 大阴影和玻璃模糊。
- 英文水印。
- 过大的 hero 标题。
- 深色侧栏和深色动态面板。
- 过多 pill CTA。
- uppercase / letter spacing 的元信息。

增加：

- 更轻的列表分隔线。
- 更像动态 feed 的首页模块。
- 贡献者头像和人名的自然露出。
- 社区来源、推荐人、编辑、组织者等“人”的信息。
- 低强调但清晰的 RSS、投稿、提交入口。

## 优先修复方向

1. 视觉 token 收敛：降低默认阴影、模糊、hover 幅度。
2. Hero 系统收敛：栏目页短标题，移除或弱化水印。
3. 卡片系统收敛：普通卡片更轻，重点卡片才突出。
4. 首页动态区改为浅色社区公告板。
5. 详情页侧栏从深色信息面板改为浅色辅助区。
6. 贡献者页加强真实成员和参与路径。
7. 中文元信息去 uppercase 化。
