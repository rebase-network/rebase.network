# Rebase 社区网站 V1 范围

> 归档说明：这是一份历史范围文档，仅供追溯背景使用；当前有效边界请查看 `docs/product/current-boundaries.md`。

## 产品目标

在这个仓库中构建新的 Rebase 社区网站，以及一个易维护、面向 Rebase 的管理工作台。

V1 应建立一个稳定的面向公众内容平台，并配套一套面向社区工作人员任务的内部工具。

首个版本应优先保证：

- 清晰的社区定位
- 能持续展示社区动态的社区媒体呈现方式
- 优秀的移动端与桌面端阅读体验
- 可持续的内容发布能力
- 对 GeekDaily 历史的可扩展处理
- 一个对运营人员友好的定制 admin，而不是通用的 headless CMS
- 在不过度建设 V1 的前提下，为未来扩展预留空间

## 产品定位

V1 是：

- 社区网站
- 内容中枢
- 对运营友好的发布平台
- 帮助读者发现 Rebase 内部动态的社区媒体体验
- 面向工作人员进行内容维护的定制 admin 工作台

V1 不是：

- 完整的社区成员产品
- 复杂的活动运营平台
- 面向读者、带账号、评论或 feeds 的社交产品

## 目标用户

- 浏览 Rebase 内容的公开读者
- 维护内容和站点配置的社区运营人员
- 未来需要明确进入社区入口的贡献者

## 范围内

### 公共页面

- 首页
- About 页面
- `Who-Is-Hiring` 页面
- 招聘详情页
- GeekDaily 列表页
- GeekDaily 详情页
- 文章列表页
- 文章详情页
- 活动列表页
- 活动详情页
- 贡献者页面
- 全局页脚内容

### 首页

首页应展示：

- 简洁的 Rebase 介绍
- 最近文章
- 最近活动
- 最近 jobs
- 最新 GeekDaily
- 通往核心社区内容的入口

### Admin 能力

工作人员应能够管理：

- 首页内容
- About 页面内容
- GeekDaily 内容
- 文章
- 活动
- jobs
- contributors
- 页脚内容
- 媒体上传
- 工作人员角色与权限
- 对敏感变更的审计可见性

### 编辑格式

V1 的内容编辑应使用：

- 结构化字段承载 metadata、taxonomy 和页面装配
- Markdown 承载长正文内容

这适用于文章正文、About 内容、活动详情内容、招聘详情内容，以及在需要时使用的 GeekDaily 期目正文。

### GeekDaily 搜索

GeekDaily 搜索属于 V1 范围。

第一版应支持按以下维度搜索：

- 标题
- 摘要
- tags
- 期号
- 日期
- 推荐条目标题

V1 搜索应优先满足务实的内容发现，而不是追求重量级全文搜索质量。

第一版实现应使用基于 Rebase 自有搜索载荷的前端搜索。

如果未来需求增长，项目后续可以接入第三方搜索服务或 plugin。

### RSS Feeds

RSS 属于 V1 范围。

第一版应包含：

- 位于 `/rss.xml` 的全站 feed
- 位于 `/geekdaily/rss.xml` 的 GeekDaily feed
- 位于 `/articles/rss.xml` 的 articles feed
- 位于 `/events/rss.xml` 的 events feed
- 位于 `/who-is-hiring/rss.xml` 的 hiring feed

全站 feed 应聚合以下公共内容中的近期内容：

- GeekDaily 期目
- articles
- events
- jobs

GeekDaily RSS items 应以期目为单位，而不是以单条条目为单位。

招聘 RSS items 应以 job 为单位，并链接到公开招聘详情页。

每个 V1 feed 默认只返回最新 3 条已发布内容。

## 范围外

以下内容明确排除在 V1 之外：

- 站内活动报名表单
- 活动报名数据收集与后台审核
- 公开用户账号
- 成员登录
- 评论
- 点赞、收藏或通知
- 多语言支持
- 复杂的编辑审批工作流
- 高级推荐系统
- 全站统一搜索
- 邮件订阅系统
- 高级 feed 分发工作流
- 成员自助资料编辑

## 活动模块边界

V1 的活动页面支持：

- 即将开始的活动
- 过往活动
- 活动详情内容
- 外部报名链接
- 报名说明

V1 不支持：

- 站内活动报名表单
- 报名管理
- 参与者导出

## 招聘模块边界

V1 的招聘页面支持：

- 公开招聘列表页
- 公开招聘详情页
- 招聘 RSS 输出
- 公开申请链接与招聘 metadata

V1 不支持：

- 站内职位申请表单
- 申请人跟踪
- 超出工作人员内容管理范围的招聘方仪表盘

建议的招聘详情内容包括：

- 公司或团队名称
- 岗位标题
- 薪资
- 是否支持远程
- 工作模式
- 写在主描述中的工作范围和职责
- 工作地点
- 联系方式或申请方式

## Admin 体验边界

V1 admin 应支持：

- 工作人员登录
- 面向任务的列表与编辑界面
- 发布前校验
- 媒体选择与上传
- 草稿、发布和归档动作
- 基于角色的访问控制
- 对敏感运营的审计可见性

V1 admin 不需要：

- 实时协同编辑
- 多步骤审批工作流
- low-code schema 编辑
- 类 page-builder 的拖拽式编排

## 成功标准

当满足以下条件时，V1 算成功：

- 核心公共页面已经实现
- 网站在桌面端和移动端上表现良好
- 工作人员可以通过定制 admin 工作台管理主要内容类型
- GeekDaily 历史可以被可靠浏览和搜索
- 媒体资源通过稳定的存储方案管理
- 架构支持长期运行与扩展
